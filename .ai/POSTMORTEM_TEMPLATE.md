## <дата> — <краткое имя инцидента>

- **Симптом** — что увидел пользователь.
- **Класс дефекта** — тип ошибки, а не только частный случай.
- **Корень** — почему произошло (код/данные/процесс).
- **Почему не поймали** — какого именно контроля не хватило (тест/гейт/мониторинг).
- **Какой тест поймал бы** — этот тест добавляется ДО фикса (красный→зелёный) и остаётся навсегда.
- **Системная правка** — что меняем в архитектуре/процессе, чтобы класс исчез (не точечная заплата).

## Правила (закреплены в DoD)

1. Прод-баг сначала воспроизводится падающим тестом, затем чинится; репро-тест остаётся в сьюте навсегда.
2. Ни один экран/модалка не уходит в Done без error boundary над содержимым (роуты покрыты `defaultErrorComponent`, ТП-131; вне-роутовые деревья — `shared/ui/components/ErrorBoundary`, ТП-172).
3. Тест красный при незамеченном `console.error`/uncaught/unhandled rejection (гейт в `src/test/setup.ts`, ТП-172 T4).

## Реестр post-mortem

### 2026-07-07 — Белый экран при открытии карточки задачи с доски

- **Симптом:** клик по карточке на доске → всё приложение исчезает (белый экран), без сообщений.
- **Класс:** router-hook в компоненте, живущем вне контекста роутера (NiceModal-модалка) → необработанное исключение рендера без boundary сносит всё дерево React. Родственен TD-015/ТП-39/ТП-131.
- **Корень:** ТП-165 добавил `useNavigate()` в `TaskCardContent`; на странице `/task/{code}` работал, в модалке (вне Router-контекста) бросал `TypeError: Cannot destructure property 'navigate' of 'useRouter(...)' as it is null`.
- **Почему не поймали:** модалка не покрыта route-level error boundary; не было теста «карточка рендерится вне роутера»; console-гейта в раннере не было; прод-мониторинга клиентских ошибок нет (сигнал — от пользователя).
- **Тест, который поймал бы:** `features/task/__tests__/taskCardOutsideRouter.test.tsx` (рендер TaskCardContent без RouterProvider) — добавлен, воспроизводил краш до фикса.
- **Системные правки:** (T1) `ErrorBoundary` + обёртка содержимого TaskCardModal; (T4) гейт console.error/uncaught в Vitest; (T7) этот шаблон и правила DoD; правило «в компонентах, переиспользуемых модалками, router-hooks запрещены — только `router.navigate`» добавлено в CLAUDE.md.
