# Стандарт «skeleton-first + прогрессивная загрузка» (ТП-185)

> Обязателен для **любой новой модалки и панели** WorkTask. Цель — убрать
> класс багов «UI ждёт сервер → долгий спиннер/пустой экран».

## Принципы

1. **Не блокировать первый рендер сетью.** Поверхность открывается мгновенно:
   скелетон + уже известные данные (из списка/маршрута/кэша). ID, код и
   заголовок задачи известны до загрузки — показываем их сразу.
2. **Прогрессивная загрузка.** Сначала первый экран; тяжёлое и вторичное
   (история, вложения, комментарии) — лениво / по раскрытию.
3. **Оптимистичные мутации.** Действие отражается сразу, синхронизация в фоне,
   при сбое — откат (пример: ТП-184, пометка уведомления прочитанным).
4. **Кэш + фоновое обновление (stale-while-revalidate).** Повторное открытие —
   мгновенно из кэша; после правок пользователя не показывать устаревшее.
5. **Единый источник правды для производных.** Счётчики/бейджи вычисляются из
   элементов, не хранятся отдельно (ТП-184).
6. **Мягкая деградация по блокам.** Не загрузился блок — понятная ошибка с
   повтором именно в нём (`ErrorBoundary`, ТП-172), никогда не белый экран и
   не вечный спиннер; честные состояния loading / ready / error / empty.
7. **Дешёвые общие справочники.** Статусы/спринты/исполнители кэшируются на
   уровне приложения и переиспользуются, а не грузятся при каждом открытии.
8. **Перф-бюджет.** Каркас < ~200–300 мс, измерять до/после.

## Общие примитивы (построены, переиспользовать)

- `shared/ui/components/Skeleton.tsx` — `SkeletonLine`, `SkeletonParagraph`,
  `SkeletonField` (тема-aware, светлая/тёмная).
- Скелетон под конкретную поверхность (пример: `features/task/TaskCardSkeleton.tsx`).
- Загрузка данных: TanStack Query (`staleTime`/`placeholderData`/фоновый
  `refetchInterval`) + состояния `isPending/isError/data`.
- Проброс известных данных: `placeholderData` из кэша списков (пример:
  `useTaskByCode` ищет задачу по коду в кэше спринтов/завершённых).
- Оптимистичная мутация с откатом: `onMutate/onError/onSettled` (пример:
  `useNotifications`).
- Граница ошибки блока: `shared/ui/components/ErrorBoundary`.

## Definition of Done (дополнение)

Ни одна новая модалка/панель не уходит в Done без: skeleton-first открытия,
прогрессивной загрузки, состояний error/empty по блокам и перф-проверки
(замер до/после). **Эталон — карточка задачи** (`TaskCardModal` +
`TaskCardSkeleton`, ТП-185): на неё ссылаться в ревью.

## Миграция существующих поверхностей

В этой задаче массовой переделки нет. Остальные модалки/панели мигрируются
отдельными атомарными задачами (по одной на поверхность) — заведены в бэклог.
