# VOICE_DEMO.md — демонстрация и краткая инструкция голосового управления

> Подсистема голосового управления WorkTask (ТП‑90..103). Веха 1 без AI.
> Встроенная подсказка в приложении генерируется из реестра команд (см.
> `features/voice/command/voiceHelp.ts`) и показывается в оверлее при непонятной
> команде — этот файл её дублирует для документации/приёмки.

## Как пользоваться

1. Нажмите плавающую кнопку микрофона (внизу справа) **или** горячую клавишу
   **Ctrl+Shift+M** (раскладко‑независимо).
2. Скажите команду естественным языком (см. примеры ниже).
3. Безопасные действия выполняются сразу; изменяющие данные — после подтверждения
   в карточке (кнопка «Подтвердить»). Опасные (завершение спринта) — всегда с
   подтверждением.
4. Кнопка‑история (часы) рядом с микрофоном — журнал команд сессии с откатом
   (кнопка «Отменить» у поддерживаемых действий, напр. смена статуса).

Голос — это способ ВВОДА поверх обычных действий: любая команда выполняется теми
же операциями, что и через интерфейс. Система ничего не выдумывает: если задача/
исполнитель/статус не распознаны однозначно — переспросит.

## Демонстрационный сценарий (по категориям ТЗ)

| Категория | Скажите | Что произойдёт |
|---|---|---|
| Создание задачи | «Создай задачу подготовить квартальный отчёт» | Создаётся задача (название·описание разделяются) |
| Создание бага | «Заведи баг не грузится доска» | Создаётся задача типа BUG |
| Смена статуса | «Переведи эту задачу в готово» | Подтверждение → статус меняется (можно откатить в журнале) |
| Смена спринта | «Перенеси эту задачу в текущий спринт» | Подтверждение → перенос |
| Смена приоритета | «Сделай эту задачу высоким приоритетом» | Подтверждение → приоритет |
| Смена исполнителя | «Назначь эту задачу на Иванова» | Подтверждение → исполнитель (по имени/фамилии) |
| Комментарий | «Прокомментируй эту задачу нужно уточнить сроки» | Добавляется комментарий |
| Открыть задачу | «Открой ТП‑90» | Переход на карточку задачи |
| Спринты | «Создай спринт Релиз 1» · «Активируй спринт …» · «Заверши текущий спринт» | Создание / активация / завершение (последнее — с подтверждением) |
| Встречи | «Создай встречу обсуждение спринта завтра в 15» | Подтверждение → встреча в календаре (разбор даты/времени локально) |
| Навигация | «Открой доску» · «Открой календарь» · «Открой настройки» | Переход в раздел |
| Уведомления | «Отметь уведомления прочитанными» | Все уведомления помечаются прочитанными |
| Свободная формулировка | «Мне нужна задачка про релиз» | Эвристика распознаёт создание → подтверждение |
| Обработка ошибок | «Переведи в готово» (без открытой задачи) | Переспрос: назвать задачу/код |
| Не распознано | «Расскажи анекдот» | Подсказка «что можно сказать» (встроенная инструкция) |
| Откат | (после смены статуса) журнал → «Отменить» | Статус возвращается на прежний |

## Что распознаётся устойчиво

- **Склонения имён**: «Ивану», «Иванову», «Петру».
- **Синонимы статусов/приоритетов**: «готово/выполнено» → Done; «срочно/критично» → High.
- **Ссылка на задачу**: «эту задачу» (открытая карточка) или код «ТП‑90».
- **Свободные фразы** без строгого триггера — через эвристику (с подтверждением).

## Границы Вехи 1 (осознанные)

- Открытая задача определяется по маршруту `/task/$code` (не по модалке поверх доски).
- Свободный интеллектуальный разбор — минимальный (эвристика); полноценный LLM
  подключается позже за тем же интерфейсом `IntentResolver` (см. `VOICE_ARCHITECTURE.md` §5).
- Разбор даты/времени встреч (ТП‑104) — относительные даты (сегодня/завтра/послезавтра,
  дни недели) + время («в 15», «в 9 утра»); абсолютные даты («5 августа») пока не разбираются.
