# AUDIT/voice-assistant.md — Глубокий аудит голосового помощника

> Оценка «как перед коммерческим релизом завтра». Дата: 2026-07-06.
> Методы: код-ревью конвейера, прогон unit-тестов (149 голосовых из 256), UI-прогон
> состояний лаунчера/оверлея; живая речь в автоматическом превью невозможна —
> сценарии распознавания оценены по коду и тестам (честное ограничение стенда).

## 1. Первый запуск и обучение — 4/5
- Первый клик по микрофону предлагает интерактивное обучение (ТП-118): проверка/калибровка
  микрофона (Web Audio, реальный уровень сигнала), spotlight-тур, практика на реальных
  сценариях с авто-открытием созданной задачи. Предложение — не чаще раза за сессию, прогресс
  персистится.
- Справочник «?» рядом с микрофоном: запуск без инструкции возможен, примеры фраз в оверлее
  и справочнике. Настройки → «Пройти обучение» — повторный вход есть.
- Слабое место: в неподдерживаемых браузерах (Firefox) фича скрыта молча (F-014, решение
  за владельцем).

## 2. Сценарии — 4/5
- Happy path: 14 команд (создание задач/багов, статус/исполнитель/приоритет/спринт,
  комментарии, спринты, навигация, уведомления, встречи с разбором дат) — рабочие
  дескрипторы поверх существующих мутаций (ADR-005), 149 unit-тестов.
- Ошибки распознавания: цепочка rule → эвристика (levenshtein/wordSimilar, склонения);
  низкая уверенность требует подтверждения даже для safe-команд (confidence gate).
- Длинная/оборванная речь: непрерывный режим с перезапуском на паузах, накопление текста,
  стоп-фраза «работаем» (ё/е, пунктуация нормализуются) — решено в ТП-111.
- Смешанные языки: не обрабатывается специально — распознавание ru-RU (принятое ограничение).
- Пунктуация/капитализация (русский): **слабейшее место** — только капитализация первой
  буквы; произнесённые «запятая»/«точка»/«с новой строки» не преобразуются → длинная
  диктовка сплошным текстом (**F-013, берём в работу**).

## 3. Обратная связь — 5/5
- Состояния видны: FAB меняет цвет/иконку (запись = красный стоп), оверлей показывает
  live-транскрипт (final+interim), обработку, подтверждение, результат, ошибку.
- Отменить можно (cancel/стоп), подтверждение перед мутациями и деструктивными действиями,
  журнал команд с undo смены статуса (ТП-103). Результат редактируем: команды создают
  сущности через обычные формы/карточки (можно править после).
- Реальная причина сбоя показывается (ТП-123, voiceErrorMessage из ответа бэкенда).

## 4. Скорость — 5/5
- Распознавание стримится браузером (interim видно сразу); резолвер локальный,
  синхронный (правила/эвристика) — от стоп-фразы до подтверждения задержка незаметна;
  сеть — только сама мутация. Замеров не требует: тяжёлых стадий в конвейере нет.

## 5. Обработка ошибок — 4/5
- Карта ошибок ASR → человеческие сообщения (not-allowed/no-speech/audio-capture/network
  с объяснением про онлайн-сервис Chrome). Отказ микрофона: pre-flight в онбординге +
  сообщение при записи. Пустой ввод: no-speech не завершает непрерывный режим.
- Восстановление без потери ввода: накопленный текст сохраняется при перезапусках
  распознавания; при ошибке команды текст остаётся в оверлее.
- Минус: диктовка в поле при сбое ASR теряет уже произнесённое? Нет — onFinish получает
  накопленное; но при `network`-ошибке cancelled=true — накопленный текст НЕ отдаётся
  в поле (теряется). Пограничный случай, отмечено в бэклоге (не блокер: тексты коротки).

## 6. Архитектура и масштабируемость — 5/5
- Конвейер за швами: SpeechProvider (замена ASR), IntentResolver (chainResolvers:
  rule → heuristic → [LLM seam, TD-017]), SlotResolver (строка→ID, анти-галлюцинация),
  ConfirmationGate, Executor, Journal. Новая команда = дескриптор в реестре
  (описание/примеры/слоты/keywords) без правки существующих компонентов (ADR-006).
- Форматирование изолировано от распознавания (localTextFormatter) — F-013 ложится
  в один модуль с unit-тестами.
- Приватность: ASR локально-браузерный, аудио не хранится; политика не формализована
  (TD-018 — при подключении облачного провайдера зафиксировать retention).

## Итог
Подсистема зрелая: архитектура — эталонная часть проекта, обратная связь и обработка
ошибок на уровне. До «коммерческого релиза завтра» не хватает:
1. **F-013 (в работу):** голосовая пунктуация в диктовке — детерминированные замены
   «запятая/точка/…» + капитализация предложений.
2. F-014 (решение владельца): видимость фичи в Firefox.
3. Бэклог: сохранение накопленного текста диктовки при network-ошибке ASR; политика
   приватности (TD-018); LLM-резолвер при появлении ключа (TD-017).
