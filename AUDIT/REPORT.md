## 3. Задачи, критерии приёмки и результат

Все задачи прошли quality gates (§7): линтер/тесты зелёные, покрытие не снизилось,
видимые изменения проверены на стенде со скриншотами «до/после».

- **ТП-128 (PR #111)** — retry-предикат вынесен в `shouldRetryQuery`; 4xx не ретраятся,
  console.log убран, staleTime 15s. Приёмка: 404 по удалённой задаче — один запрос
  вместо четырёх, консоль чистая. +3 теста.
- **ТП-130 (PR #112)** — `TaskCardLoadError`; при 404 карточка показывает «задача не
  найдена» вместо вечного спиннера. Приёмка на стенде: клик по уведомлению удалённой
  ТП-126 → сообщение + «Закрыть». +2 теста.
- **ТП-131 (PR #113)** — `NiceModal.Provider` перенесён в `createRouter({ Wrap })`,
  добавлен `defaultErrorComponent` (RouteErrorFallback). Приёмка: карточка-модалка с
  router-хуками открывается; краш рендера даёт восстановимый экран, не белый.
- **ТП-132 (PR #114)** — `navigate()` из рендера AuthPage → в `useEffect`. Приёмка:
  логаут→логин→редирект, консоль без 6 ошибок «Cannot update…».
- **ТП-133 (PR #119)** — единый `ConfirmDialog` (promise/NiceModal), заменены все 7
  `window.confirm`. Приёмка: удаление задачи → тематический диалог; «Отмена» не удаляет.
- **ТП-134 (PR #120)** — адаптив шапки: боковой блок 56px при ≤640px, компактная «+»,
  скрытое имя проекта. Приёмка: 375px — колокольчик/аватар в вьюпорте; 1440px без изменений.
- **ТП-135 (PR #118)** — голосовая пунктуация `applySpokenPunctuation` (запятая/точка/…/
  новый абзац → знаки, капитализация). +11 тестов.
- **ТП-136 (PR #117)** — кнопка «Создать» не блокируется по !isValid; пустое название →
  видимая ошибка у поля. Приёмка на стенде со скриншотом.
- **ТП-137 (PR #115)** — сообщение «Не найден статус …» вместо «Не найдена задача …».
  +1 backend-тест.
- **ТП-129 (PR #116)** — устранён единственный ESLint warning (useTour exhaustive-deps).

## 4. Архитектурные решения (мини-ADR)

- **ADR-A: NiceModal внутрь роутера через `Wrap` (ТП-131).** Проблема: Provider вне
  RouterProvider — router-хуки в модалках падали (TD-015, класс белых экранов).
  Альтернативы: (1) `window.location` вместо router-хуков в каждой модалке — обходной
  путь, множит хрупкость; (2) поднять RouterProvider выше — ломает порядок провайдеров.
  Выбрано: официальная опция TanStack `Wrap` — модалки в дереве роутера без обходов.
  Основание: §2 «целостность архитектуры».
- **ADR-B: корневой `defaultErrorComponent` (ТП-131).** Краш рендера ронял всё приложение.
  Выбран route-level error boundary TanStack (восстановление через `router.invalidate()`)
  как идиоматичный для стека, вместо самописного React ErrorBoundary поверх RouterProvider.
- **ADR-C: `retry: false` на by-code запросе + глобальный no-retry для 4xx (ТП-128/130).**
  4xx детерминированы; для поиска задачи по коду 404 — нормальный терминальный исход.
  Глобальный предикат чинит шум, локальный `retry:false` делает ТП-130 самодостаточным.
- **ADR-D: голосовая пунктуация только правилами (ТП-135).** Соответствует ADR голоса
  (форматирование изолировано от распознавания, без AI). Альтернатива — LLM-пунктуатор —
  отвергнута: избыточно, требует ключа (TD-017), недетерминированно. Паттерн Google
  voice typing.
- **ADR-E: единый `ConfirmDialog` (ТП-133).** `window.confirm` заменён promise-based
  NiceModal-диалогом в стиле приложения — консистентность UI (§2, п.3) и совместимость с
  ADR-A (NiceModal.show работает независимо от места Provider).
